01Premise

A whole tiny OS in the browser tab.

Tidepool is a terminal-first portfolio rendered with xterm.js. Every page is a shell command — help for the catalog, cat about.md for the bio, repos for a live table of GitHub repositories.

A virtual Unix filesystem lets you cd projects && ls -l. Tab completion, command history, ANSI 24-bit color, and a boot animation make it feel like a real shell.

03Quickstart

Clone, install, run.

Standard Vite project. The dev server prints a local URL (usually http://localhost:5173).

# Prereqs: Node.js 18+
$ git clone https://github.com/Real-Fruit-Snacks/Tidepool.git
$ cd Tidepool
$ npm install
$ npm run dev                   # → http://localhost:5173

# Production build (hashed assets, xterm.js split into its own chunk)
$ npm run build
$ npm run preview

# Inside the running terminal
$ help
$ neofetch
$ cat about.md
$ repos

05Architecture

Boot, then shell.

Boot sequence runs first; the shell takes over once it finishes. All output flows through the formatter for consistent ANSI styling and box-drawn tables.

src/
main.js              // Entry point
shell.js             // Input · prompt · execution
terminal.js          // xterm.js setup with addons
filesystem.js        // Virtual filesystem
formatter.js         // ANSI · box drawing · tables
github.js            // GitHub data loader
history.js           // Persistent command history
autocomplete.js      // Tab completion
boot.js              // Boot animation
permalink.js         // URL hash read/write
theme.js             // Catppuccin Mocha palette
commands/            // Per-command implementations
06Closing

Local data, no tracking.

GitHub data is fetched by a daily Action and committed to public/data/github.json — no API tokens reach the frontend, no runtime API calls. Command history and theme persist in localStorage on your device only.

Tested across current Chrome, Firefox, Safari, and Edge. Mobile-friendly with a touch keyboard toggle.
